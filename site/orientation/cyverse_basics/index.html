    Connecting Cyverse to GitHub

    Log in to Cyverse

    1. Go to the Cyverse user account website https://user.cyverse.org/

    1. Click Sign up (if you do not already have an account)

    1. Head over to the Cyverse Discovery Environment https://de.cyverse.org, and log in with your new account.

    You should now see the Discovery Environment:

    1. We will give you permissions to access the Hackathon app. If you haven't already, let us know that you need access

    Open up an analysis with the hackathon environment (Jupyter Lab)

    Set up your GitHub credentials

    If you would prefer to follow a video instead of a written outline, we have prepared a video here:

    1. From Jupyter Lab, click on the Git Extension icon on the left menu: Raw photo location: jupyterlab.svg

    2. Click Clone a Repository and Paste the link to the cyverse-utils https://github.com/CU-ESIIL/cyverse-utils.git and click Clone: Raw photo location: clone.svg

    3. You should now see the cyverse-utils folder in your directory tree (provided you haven't changed directories from the default /home/jovyan/data-store Raw photo location: cyverse-utils.svg

    4. Go into the cyverse-utils folder: Raw photo location: click_cyverse_utils.svg

    5. open up the create_github_keypair.ipynb notebook if you prefer Python or the 'create_github_keypair.R' script if you prefer R by double-clicking and then select the default 'macrosystems' kernel: Raw photo location: open_cyverse_utils.svg

    6. Now you should see the notebook open. Click the play button at the top. You will be prompted to enter your GitHub username and email: Raw photo location: script_1.svg

    Raw photo location: username.svg

    Raw photo location: email.svg

    1. You should now see your Public Key. Copy the WHOLE LINE including ssh-ed25519 at the beginning and the jovyan@... at the end Raw photo location: key.svg

    2. Go to your GitHub settings page (you may need to log in to GitHub first): Raw photo location: settings.svg

    3. Select SSH and GPG keys Raw photo location: ssh.svg

    4. Select New SSH key Raw photo location: new_key.svg

    5. Give your key a descriptive name, paste your ENTIRE public key in the Key input box, and click Add SSH Key. You may need to re-authenticate with your password or two-factor authentication.: Raw photo location: paste_key.svg

    6. You should now see your new SSH key in your Authentication Keys list! Now you will be able to clone private repositories and push changes to GitHub from your Cyverse analysis! Raw photo location: final.svg

    NOTE! Your GitHub authentication is ONLY for the analysis you're working with right now. You will be able to use it as long as you want there, but once you start a new analysis you will need to go through this process again. Feel free to delete keys from old analyses that have been shut down.
